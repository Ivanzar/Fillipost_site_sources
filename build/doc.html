Документация

Создание проекта

Linux:

$ fillipost-cli create <project path>

Windows:

$ fillipost-cli.bat create <project path>

Сборка проекта

Linux:

$ fillipost-cli build <project path>

Windows:

$ fillipost-cli.bat build <project path>

Команды
build <project path> [flag] Сборка проекта - генерация статических страниц. Флаги:
  --resourcesСборка только ресурсов (папка resources)
  --post [post file]Сборка отдельного поста (папка posts)
  --posts-onlyСборка только постов (папка posts)
create <project path> Создание нового проекта.
clean <project path> Удаляет деррикторию project path/build.

Строение дериктории проекта

  • project path  нзавние дерриктории проекта
    • build   дериктороия со сгенерированными старницами и ресурсами
    • patterns   дериктория с шаблоанми
    • posts   дериктория со старницми в Markdown формате(назвние файлов должно оканчиваться на .md)
    • resources дерриктория с ресурасами проекта
    • snippets   шаблоны для вставки в код файлов дериктории patterns

Шаблон старниц в дериктории post

---

{
  "post": {
    "name": string,
    "author": string,
    "description": string,
    "pattern": string,
    "date": int
  },
  "variable1": ,
  "variable2": ,
  "variable3": ,
  "variableN": 
}

---

Текст в Markdown формате

Описание:

--- - начало заголовка

"post" - обязательное поле

"name" - название старницы, обязательно

"author" - имя автора, не обязательно

"description" - описание поста, не обязательно

"pattern" - шаблона для страницы(в папке patterns), обязательно

"date" - дата unix epo time, не обязательно

"variable" - другие поля, не обязательно

--- - конец заголовка
